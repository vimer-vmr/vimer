Quick start

Installation#

Download the ViMeR setup file for Windows or macOS from the releases page of the gitlab project. The setup file will install ViMeR when you execute the file. ViMeR will open after the installation.

macOS: Unidentified developer

The executable for macOS is currently not digitally signed. As a result macOS will show a warning message when you try to open the application. Please take a look at this Apple support page for instructions on how to allow the application to run on your mac.

Before testing#

Before you can start testing participants you have to specify the settings for the application and survey in a configuration file. Here we provide you with an example configuration file. All available response formats are used in this example. For details about the settings, please take a look at the reference manual page. To start configuring ViMeR, download the example configuration file and edit it using an editor like Visual Studio Code with the YAML plugin.

config.yml
general:
  timing:
    type: relative
    interval: 2
    segments: 5
  labels:
    previous: Previous
    next: Next
    submit: Submit
    play: Play
    replay: Replay
  ui:
    showReplayButton: true
    showPreviousButton: true
    videoWidth: 50
    image: assets/uni1.png

instructions:
  title: Instructions
  text: A short introduction video about big buck bunny will be shown. Please pay attention to everything you see. The movie will be interrupted and questions about the most recent movie fragment will be asked.

outro:
  title: Thanks for participating
  text: Please go to the experiment instructors. They will guide you to the next part of this experiment.

survey:
- page: Scene
  description: In the previous movie segment ...
  questions:
  - question: I saw an animal that looks most like a
    label: scene_animal
    type: radio
    options: 
      - Bird
      - Bunny
      - Tiger
      - None
  - question: I think they tried to show...
    label: scene_situation
    type: text
- page: Feelings
  description: In the previous movie segment ...
  questions:
  - question: I felt ... happy.
    label: feelings_happy
    type: slider
    options:
      show-value: false
      min: 0
      max: 100
      labels:
        left: not at all
        center: ""
        right: very
  - question: I felt ... sad
    label: feelings_sad
    type: likert
    options:
      show-value: true
      values:
      - [ -3, not at all]
      - [ -2, ]
      - [ -1, ]
      - [ 0, ]
      - [ 1, ]
      - [ 2, ]
      - [ 3, very]
- page: Feelings (2)
  description: In the previous movie segment ...
  questions:
  - question: I felt.
    label: feelings_grid
    type: grid
    options:
      size: 7
      labels:
        left-top: Stress
        center-top: High arousal
        right-top: Excitement
        right-center: Pleasant feelings
        center-bottom: Sleepiness
        left-bottom: Depression
        right-bottom: Relaxation
        left-center: Unpleasant feelings

During testing#

After starting the program you will see the setup screen in which you will have to specify:

  • a participant code which is a unique identifier for your participant
  • the video file you want to show to the participant. In the example below a fragment of big buck bunny was used.
  • a settings file that contains the survey and other settings
  • the output location where the resulting data will be stored.

After clicking the Next button on the setup pane, the instructions are shown.

After clicking the Next button on the instructions pane, the participant has to click Play to start the first segment of the video. After this segment the survey is shown.

After filling out the survey, the next segment will play, the next survey will be shown, and this sequence continues until the end of the video file.

At the end the outro pane is shown. This will show the text from the outro section in the configuration file.
