Reference

Configuration file format#

The configuration file is in the YAML format and contains multiple sections:

  • general
  • instructions
  • outro
  • survey

A YAML file is a text file with the extension .yml. The basic syntax of YAML is to use key-value pairs in the format key: value. In YAML, spaces are used to indicate nesting. Some YAML fields take unnamed vectors as their value. You can specify an element of the vector by adding a new line and -.

All keys and values are required

All keys and values in the general, instruction and outro section are required. All keys in a question type are required for that question type. In some cases if you don't want to assign a value, you can assign an empty value "".

General#

Timing#

The video will be divided into segments. The researcher can choose to show a fixed number of segments (absolute) or segments of a fixed duration (relative). To do this absolute or relative have to be assigned to type. If absolute is chosen, the value of interval determines the duration of the movie segments in seconds. The value of segments is ignored. If relative is chosen, the value of segments specifies the number of segments the participant will rate. The value of interval is ignored.

In the example below the movie will be split in 10 parts with an equal duration. After each part the participant has to fill in a survey. 

general:
  timing:
    type: relative
    interval: 2
    segments: 10

Number of segments using absolute timing

In case of absolute timing only intervals of the entire selected duration are included, meaning that selecting an absolute interval of 30 seconds for a video of 5 minutes and 20 seconds results in the final 20 seconds not being shown to participants.

Labels#

The labels on the buttons to go to the previous or next page of questions, to submit the questions, play or replay the video segment can be changed by the researcher. In the example below the Dutch translation of the words will be shown. 

general:
  labels:
    previous: Vorige
    next: Volgende
    submit: Opslaan
    play: Speel af
    replay: Herhaal

UI#

Other user interface elements can be customized: 

general:
  ui:
    showReplayButton: true
    showPreviousButton: true
    videoWidth: 50
    image: assets/uni1.png
showReplayButton has a boolean value (true/false) that determines if the replay button for the video will be visible when the video paused

showPreviousButton has a boolean value (true/false) that determines if the previous button will be visible on every page of the survey

videoWidth is the percentage of the width of the ViMeR window that will be filled with the video

image contains the location of the logo you want to show. Built-in images are assets/uni1.png, assets/uni2.png and assets/uni3.png.

Instructions#

You can add some instructions for the participants that will be shown after the setup and before the actual video-mediated recall task starts. 

instructions:
  title: Welcome to the experiment
  text: Please watch the video fragments <b>carefully</b> and answer the subsequent questions.

        When you finished answering the questions, click Submit.

The instruction page contains a title in a larger font and text that can span multiple lines. Formatting can be done using HTML tags like <i>italic</i>, <b>bold</b>, etc.

Outro#

After the last survey has been completed, a thank you message or instructions for the participant will be shown again. 

outro:
  title: Thanks
  text: Please return to the experiment leader
The outro contains a title in a larger font and text that can span multiple lines. Formatting can be done using HTML tags like <i>italic</i>, <b>bold</b>, etc.

Survey#

Page#

Surveys are divided in one or more pages of questions. Every page can have a title and a description (Optional, can also be left blank by using ""). 

survey:
- page: Yourself
  description: During the movie segment I just saw, I remember I felt...
  questions:
  - question: ...
- page: Partner
  description: During the movie segment I just saw, I remember my partner felt...
  questions:
  - question: ...
The title is rendered using a larger font. The description can be used to show i.e. the start of a question that is the same for every question on the page. The questions key should be followed by one or more questions as shown in the questions section.

Questions#

survey:
- page: ""
  description: During the movie segment I just saw, I remember I felt...
  questions:
  - question: pressure from my partner to not feel stressed or sad.
    label: pressure_sad
    type: slider
    options:
      show-value: true
      min: 0
      max: 100
      labels:
        left: not at all
        center: ""
        right: very
A list of questions can be specified in each page. Every question has a label, a type and options. The label is used to name the results column in the data file. The label of every question has to be unique. It should not contain spaces. In the type field you choose the response type. options defines the details about the response type.

Slider#

A slider answer format can be presented to the participant. min and max define the extreme values. show-value has a boolean value where true will show the selected value on top of the slider. The labels at the left, the center and the right anchor of the slider can be customized. In the response file you will find a numeric value between and including the specified min and max.

survey:
- page: ""
  description: During the movie segment I just saw, I remember ...
  questions:
  - question: I felt ... happy.
    label: emotion_happy
    type: slider
    options:
      show-value: true
      min: 0
      max: 100
      labels:
        left: not at all
        center: ""
        right: very

Likert#

A likert scale answer format can be shown to a participant showing round buttons that can be selected and can have a label at their bottom. If show-value has the value true, the value will be shown on top of the round button. The value that is specified for the round button the participant has chosen is stored in the response file. 

survey:
- page: ""
  description: During the movie segment I just saw, I remember ...
  questions:
  - question: I felt ... sad
    label: feeling_sad
    type: likert
    options:
      show-value: true
      values:
      - [ -3, not at all]
      - [ -2, ]
      - [ -1, ]
      - [ 0, ]
      - [ 1, ]
      - [ 2, ]
      - [ 3, very]

Text#

If you want participants to fill in an open text question you can use the text response type. There are no options available to customize this response type. 

survey:
- page: ""
  description: During the movie segment, ...
  questions:
  - question: I think they tried to show...
    label: scene
    type: text

Radio buttons#

If you need a participant to choose exactly one response from a list you can present him with the radio response type. options contains a list of values that will be presented in a vertical list. The response file will contain the chosen value. 

survey:
- page: ""
  description: During the movie segment, ...
  questions:
  - question: I saw an animal that looks most like a ...
    label: food_craving
    type: radio
    options:
    - Bird
    - Bunny
    - Tiger
    - None

Grid#

A grid can be presented to the participant. The horizontal and vertical number of squares can be specified in size. A label for every corner and horizontal and vertical center position can be specified as you can see in the example below. In the results file you will find x=1,y=1 if the participant chose the left top response and x=7,y=7 for the right bottom response in case size is 7. 

survey:
- page: ""
  description: Please answer the questions below
  questions:
  - question: In the previous movie segment ...
    label: affect_grid
    type: grid
    options:
      size: 7
      labels:
        left-top: Stress
        center-top: High arousal
        right-top: Excitement
        right-center: Pleasant feelings
        center-bottom: Sleepiness
        left-bottom: Depression
        right-bottom: Relaxation
        left-center: Unpleasant feelings

Results file format#

A results file is created when you start the video-mediated recall task. Data will be added every time the participant completes a survey. The format of the file name of the data file is a combination of the participant code, the file name of the configuration file and the file name of the movie file separated by an underscore.

The CSV file contains these columns:

  • participantCode: The participant code that was entered in the setup pane
  • responseTimestamp: date and time the survey was written to the data file in the UTC time zone
  • segmentStartTimestamp: start time of the movie segment in seconds from the beginning of the movie clip
  • segmentEndTimestamp: end time of the movie segment in seconds from the beginning of the movie clip
  • ...: one or more columns named by the label field in the configuration file storing the data from response to the question
